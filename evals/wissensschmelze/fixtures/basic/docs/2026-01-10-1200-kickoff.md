# Projekt Kickoff (Notizen)

Wir bauen einen Benachrichtigungsdienst. Erste Grundsatzentscheidung:

- **Polling**: Clients fragen alle 30s den Server nach neuen Events.
- Begründung: einfach umzusetzen, keine offene Verbindung nötig.

Annahme: Die Last bleibt klein (< 100 Clients), Polling reicht locker.

Datenhaltung: erstmal SQLite, das reicht für den Prototyp.
