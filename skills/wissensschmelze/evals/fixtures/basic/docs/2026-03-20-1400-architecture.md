# Architektur-Stand (März)

Aktueller Aufbau:

- **Zustellung:** Webhooks mit exponentiellem Backoff, max. 5 Retries.
- **Datenhaltung:** Wechsel von SQLite auf PostgreSQL, weil mehrere Worker
  parallel schreiben.
- **Queue:** Events landen in einer Redis-Queue, Worker arbeiten sie ab.

Offen: Dead-Letter-Handling für Webhooks, die nach 5 Retries weiter scheitern.
